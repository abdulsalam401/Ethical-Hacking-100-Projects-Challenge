# 🔍 Project 34: IoT Device Scanner & UPnP/SSDP Security Auditor

Part of the **Ethical Hacking 100 Projects Challenge**. This project is a network scanning and discovery utility designed to identify local IoT (Internet of Things) devices using SSDP (Simple Service Discovery Protocol), map UPnP (Universal Plug and Play) services, audit them for configuration risks, and output formatted HTML security reports.

---

## 📖 Overview

Many smart devices and home routers expose UPnP/SSDP endpoints by default. This project acts as a network security auditor to:
1. **Discover Devices**: Query the local network multicast group address (`239.255.255.250:1900`) for active UPnP root devices.
2. **Profile Services**: Request and parse XML device descriptors (manufacturer info, model details, friendly names, and service listings).
3. **Assess Risks**: Audit exposed services (such as `WANIPConnection` which allows external port mappings or exposed `MediaServer` endpoints) and alert on common vendor default credential configurations.

---

## 🛠️ Features

- **SSDP Multicast Discovery**: Sends UDP discovery queries to map local network nodes.
- **XML Schema Profiling**: Requests device specifications and parses service types, IDs, and control endpoints.
- **Security Check Matrix**: Checks for exposure risks, vulnerable settings, and vendor default credential configurations.
- **Multi-Threaded Execution**: Parallelizes HTTP requests and device probing using `ThreadPoolExecutor`.
- **HTML Report Generation**: Automatically compiles findings, severity ratings, description indexes, and remediation guidelines into a responsive HTML dashboard.

## ⚙️ Setup & Installation

Ensure you have Python 3 installed. The tool depends on `requests` and `colorama` (optional, for CLI styling).

Install requirements:
```bash
pip install requests colorama
```

---

## 🚀 Usage

### 1. Basic SSDP Scan
Run the scanner to discover UPnP devices and generate an HTML report:
```bash
python upnp_scanner.py
```

### 2. Service-Profiling Scan
Extract more detailed vendor attributes, UDIDs, and check for specific risk signatures:
```bash
python upnp_scanner2.py --timeout 5 --output upnp_report.html
```

### 3. Full-Network Probing Scan
Scan the local subnet via ping sweeps and correlate with discovered UPnP endpoints:
```bash
python upnp_scanner_enhanced.py
```

## 🛡️ Defensive Remediation

To secure your network against UPnP vulnerabilities:
1. **Disable UPnP Globally**: Turn off UPnP on router/gateway devices if not explicitly required.
2. **Segment IoT Networks**: Isolate smart TVs, cameras, and IoT controllers on a separate VLAN from workstations containing sensitive data.
3. **Firmware Security Updates**: Regularly patch IoT firmware to address vendor-specific UPnP stack vulnerabilities.
4. **Ingress Filtering**: Ensure the router firewall drops external SSDP (port 1900) requests.
