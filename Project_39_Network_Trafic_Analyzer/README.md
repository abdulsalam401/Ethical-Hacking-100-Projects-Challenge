# Project 39: Network Traffic Analyzer

Part of the **Ethical Hacking 100 Projects Challenge**, this project is a live packet analysis and intrusion-detection lab for educational and authorized network monitoring. It captures traffic with Scapy, tracks protocol usage, highlights suspicious activity patterns, and exports an HTML summary report.

## Overview

The folder includes two related analyzer implementations:

1. **traffic_analyzer.py** - a lightweight live dashboard that focuses on protocol distribution, top source IPs, and simple SYN-flood style anomaly detection.
2. **traffic_analyzer1.py** - a more detailed packet analyzer that expands coverage with connection tracking, DNS query tracking, port-scan detection, and report generation.

Both scripts are designed for lab use and require elevated privileges to sniff live network traffic.

## Features

- Real-time packet capture with Scapy.
- Protocol classification for TCP, UDP, ICMP, DNS, HTTP, HTTPS, and SSH traffic.
- Live dashboard output with packet counters and top talkers.
- Basic intrusion heuristics such as SYN flood and port scan detection.
- HTML report generation for offline review.
- Local loopback simulation support for safe testing in a lab.

## Files

- [traffic_analyzer.py](traffic_analyzer.py): Dashboard-style analyzer that produces `traffic_summary_report.html`.
- [traffic_analyzer1.py](traffic_analyzer1.py): Expanded analyzer variant with deeper packet statistics and `traffic_report.html` output.
- [traffic_report.html](traffic_report.html): Sample report generated by the analyzer.
- [traffic_summary_report.html](traffic_summary_report.html): Compact summary report generated by the analyzer.

## Requirements

- Python 3.8 or newer.
- `scapy`
- `colorama` is optional, but it improves terminal output.
- `netifaces` is optional for auto-detecting the default interface in `traffic_analyzer1.py`.

Install the dependencies with:

```bash
pip install scapy colorama netifaces
```

## Usage

Run the dashboard analyzer on a specific interface you are authorized to monitor:

```bash
sudo python3 traffic_analyzer.py --interface lo --duration 15
```

For the more detailed variant:

```bash
sudo python3 traffic_analyzer1.py
```

Useful options from the analyzer scripts include:

- `--interface` to choose the sniffing interface.
- `--duration` to limit capture time.
- `--output` in the alternate analyzer variant to rename the HTML report.

## Output

- Console output shows protocol counts, active talkers, and anomaly alerts.
- Reports are written to `traffic_summary_report.html` or `traffic_report.html`, depending on the script used.
- When run on `lo`, the analyzer can generate safe local traffic for lab verification.

## Ethical Use

This project is for defensive learning, lab validation, and authorized monitoring only. Do not capture traffic on networks you do not own or have explicit permission to inspect.
