🎯 WHAT IS THIS PROJECT?
═══════════════════════════════════════════════════════════════════════════════

A REVERSE SHELL is a remote command execution tool used in authorized 
penetration testing and security research.

Key Difference from Normal Shells:
  • Normal Shell (SSH):    You connect TO the victim
  • Reverse Shell:         Victim connects BACK to you ◄── That's "reverse"!

This Project Includes:
  ✓ client.py     - Payload to run on victim's machine
  ✓ listener.py   - Command & Control server on attacker's machine
  ✓ gui_visualizer.py - Interactive visual guide (NEW!)
  ✓ Complete documentation with diagrams and examples

🏗️  HOW THE ARCHITECTURE WORKS
═══════════════════════════════════════════════════════════════════════════════

Simple Overview:

  Victim Machine                      Attacker Machine
  ┌──────────────────┐                ┌──────────────────┐
  │  client.py       │    ◄────────►  │ listener.py      │
  │  (runs here)     │   Encrypted    │ (controls here)  │
  │                  │   TCP:4444     │                  │
  │ Executes         │                │ Sends commands   │
  │ commands &       │                │ Gets output      │
  │ sends output     │                │                  │
  └──────────────────┘                └──────────────────┘


Command Flow:

  1. Attacker types: whoami
            ↓
  2. listener.py encrypts: "whoami" (AES-256)
            ↓
  3. Sends over TCP (encrypted)
            ↓
  4. client.py receives (encrypted)
            ↓
  5. client.py decrypts: "whoami"
            ↓
  6. Executes: subprocess.run("whoami")
            ↓
  7. Gets output: "admin"
            ↓
  8. Encrypts: "admin" (AES-256)
            ↓
  9. Sends over TCP (encrypted)
            ↓
  10. listener.py receives (encrypted)
            ↓
  11. Decrypts: "admin"
            ↓
  12. Displays: admin
            ↓
  Ready for next command!


🔐 WHY ENCRYPTION?
═══════════════════════════════════════════════════════════════════════════════

Without Encryption:
  ✗ Network admin can see all commands
  ✗ Network monitoring tools capture traffic
  ✗ Passwords, credit cards, data visible
  ✗ Easy to detect the shell activity

With AES-256 Encryption:
  ✓ Commands hidden from eavesdropping
  ✓ Output hidden from network monitoring
  ✓ Data confidentiality protected
  ✓ Harder (but not impossible) to detect

The Challenge:
  • We encrypt the DATA (commands & output)
  • But we DON'T hide that a connection exists
  • So IDS can still detect WHEN you're using it
  • But NOT WHAT commands you're running

🆘 QUICK TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

"Connection refused"
  → Listener not running? Start it first
  → Wrong IP? Check listener IP address
  → Firewall blocking? Check port 4444

"Decryption error"
  → Passphrases don't match
  → Both must use same --passphrase value
  → Default is "ultraprohacker"

"Module not found: cryptography"
  → Run: pip install cryptography

"Command timed out"
  → Command taking more than 30 seconds
  → Client enforces 30-second timeout
  → Avoid long-running commands

"Passphrase must match listener"
  → Double-check spelling
  → Default passphrase is "ultraprohacker"
  → If custom, write it down and use exactly
