💡 UNDERSTANDING THE PROJECT
═══════════════════════════════════════════════════════════════════════════════

WHAT IS A REVERSE SHELL?

Normal Connection:
  You (attacker) → connect to → Victim
  (You initiate)

Reverse Shell:
  Victim → connects back to → You (attacker)
  (Victim initiates) ◄── That's why it's "reverse"!

HOW THIS ONE WORKS:

  1. You start listener.py (your machine)
  2. Attacker sends client.py to victim
  3. Victim runs client.py
  4. client.py connects to listener.py
  5. listener.py accepts the connection
  6. You now have a shell on the victim!

WHY ENCRYPT?

  • Commands are hidden from network monitoring
  • Output is unreadable if intercepted
  • More sophisticated than plain-text shells
  • Still detectable by behavior (connections every 5s)
